# 计价鸭（Priceduck）- 离线比价 App

一款面向日常购物的离线比价工具，采用 Flutter + Cupertino 风格打造。所有数据保存在手机本地，不上传、不联网。

## 设计目的

在超市购物时，同一件商品往往以不同包装规格出售（如 2kg 装与 500g 装），不同超市的定价方式也各不相同。直接比较「总价」或凭直觉估算「单价」，既容易算错，也难以判断到底哪一款更划算。

计价鸭的设计目标，是让用户**用手机随手记录一笔物价**，由 App 自动把五花八门的包装规格换算成统一的基准单价，从而一眼看清「谁更便宜」：

- **统一基准**：无论录入的是 g / kg / ml / L / 个，App 都会归一化为 元/kg、元/L、元/个 三种基准单价，精度统一保留 2 位小数。
- **离线优先**：数据存放于手机本地 SQLite 数据库，无需注册账号、无需联网，保护隐私、随用随记。
- **两种比价视角**：
  - 按「物品」查询 —— 横向比较同一商品在各超市的单价，列表按单价从低到高排列，最划算的排在最前；
  - 按「物品 + 超市」查询 —— 查看某商品在单一超市的历史单价走势（折线图），掌握价格波动。

## 使用方式

App 底部有两个标签页：「记录」与「查询」。

### 一、记录物价（记录 Tab）

1. 在表单中填写：
   - **超市**（选填）：如「山姆」「盒马」。留空表示不记录超市。
   - **商品**（必填）：如「鸡蛋」。
   - **价格**：该包装的售价，例如 `20`。
   - **数量**：包装规格中的数值，例如 `2`。
   - **单位**：点击「单位」右侧文字弹出选择器，可选 `g` / `kg` / `ml` / `L` / `个`。
2. 填写过程中，表单下方会**实时预览归一化单价**（如 `¥10.00/kg`），方便录入时核对。
3. 点击「保存」：
   - 系统校验「商品不为空、价格 > 0、数量 > 0」，校验通过弹出「保存成功」提示并清空表单；
   - 校验不通过会在对应字段下显示错误提示。
4. 表单下方以**倒序（最新在上）**展示全部历史记录，每条显示「商品 · 超市」、记录时间和归一化单价。
5. 点击任意一条记录，从底部弹出「记录详情」：
   - 可点击右上角「编辑」修改超市 / 商品 / 价格 / 数量 / 单位；
   - 可点击「删除记录」移除该条（删除前二次确认）。

### 二、查询比价（查询 Tab）

1. 顶部两个下拉框可**输入也可选择**（从历史录入中自动匹配）：
   - **物品**（必填）：要比价的商品；
   - **超市**（选填）：限定具体门店。
2. 点击「搜索」：
   - 若未填写物品，提示「请输入要查询的物品」；
   - 点击「重置」清空筛选条件。
3. 查询结果分两种情况：
   - **仅选物品** → 列表形式展示该商品在所有记录中的单价，**按归一化单价从低到高排列**，最划算的排最前；每条显示原始「价格 / 数量+单位」与归一化单价。
   - **物品 + 超市** → 折线图展示该商品在此超市的**单价随时间走势**，下方附明细列表；当记录不足 2 条时提示「数据点不足，至少需要 2 条记录才能绘制图表」。

### 数据说明

- 所有记录保存在本机 SQLite 数据库，卸载 App 会一并清除。
- 当前版本不含云端同步、导出备份与统计报表，专注核心的「记录—比价」闭环。
