# 计价鸭 V1.1 — APK Action 构建失败诊断与修复

**日期**：2026-07-18
**触发**：GitHub Action `Build Flutter APK` 在 `flutter build apk --release` 阶段失败
**结论**：3 类问题叠加，其中 1 个是根因（引发 90% 报错），2 个是独立小 bug

---

## 根因 #1（主因）：drift 三个源文件缺 `part` 指令

`@DriftDatabase` / `@DriftAccessor` / `@DataClassName` 生成的符号
（`PriceRecordData`、`priceRecordDao`、`priceRecords`、`_$AppDatabase`、`_$PriceRecordDaoMixin`）
必须靠 `part '*.g.dart';` 挂回源文件。三个文件原先**都没写 `part`**，
导致 CI 中 `build_runner`（workflow 第 38–39 行确有执行）生成的代码接不进来，
所有 `not found` / `isn't a type` / `doesn't conform to the bound` 全是这一处连锁出来的。

**修复**：在三个文件加 `part` 指令
| 文件 | 新增 |
|------|------|
| `lib/database/tables/price_records.dart` | `part 'price_records.g.dart';` |
| `lib/database/app_database.dart` | `part 'app_database.g.dart';` |
| `lib/database/daos/price_record_dao.dart` | `part 'price_record_dao.g.dart';` |

> 注：`build_runner` 步骤已存在于 CI，无需改 workflow；加完 `part` 后重新跑即生成对应 `.g.dart`（已被 .gitignore 忽略，属正常）。

## 根因 #2（独立 bug）：`record_page.dart` 非 const 构造塞进 const 上下文

第 31 行 `const Padding(...)` 内包了 `Container(color: CupertinoColors.systemGrey.withAlpha(76))`，
`withAlpha` 是方法调用、非编译期常量，不能出现在 `const` 表达式中。

**修复**：去掉该 `Padding` 的 `const`（其内 `Container` 随之变为非 const，合法）：
```dart
Padding(
  padding: const EdgeInsets.symmetric(horizontal: 16),
  child: Container(height: 1, color: CupertinoColors.systemGrey.withAlpha(76)),
),
```

---

## 根因 #3（独立 bug）：`record_form.dart` 的 `CupertinoFormRow(title:)` 参数名错

`CupertinoFormRow` 的标签参数名是 **`prefix`**，不是 `title`（5 处全错）。

**修复**：`CupertinoFormRow` 的 `title: const Text(...)` → `prefix: const Text(...)`。
（注意：`CupertinoAlertDialog` 的 `title:` 是正确的，未改动 —— `record_detail_sheet.dart` / `query_page.dart` 两处 `title:` 属此类，保留。）

---

## 根因 #4（独立 bug）：`price_record_dao.dart` 的 `OrderingTerm(desc:/asc:)` API 误用

drift 的 `OrderingTerm` 没有 `desc`/`asc` 命名参数，正确写法是
`OrderingTerm(expression: <列>, mode: OrderingMode.desc/asc)`。5 处全错。

**修复**：
- `OrderingTerm(desc: t.createdAt)` → `OrderingTerm(expression: t.createdAt, mode: OrderingMode.desc)`
- `OrderingTerm(asc: t.product)`   → `OrderingTerm(expression: t.product, mode: OrderingMode.asc)`
- `OrderingTerm(asc: t.store)`     → `OrderingTerm(expression: t.store, mode: OrderingMode.asc)`

## 级联误报（无需改代码，#1 修好后自动消失）

- `record_repository.dart:55` 「Enums can't be instantiated」：
  `ProductUnit.fromSymbol` 是 `extension ProductUnitX` 上的 **static 方法**，本身正确；
  该报错是 `PriceRecordData` 未解析导致的级联误报，#1 修复后消失。
- `priceRecords` / `product` / `store` / `priceRecordDao` 「not defined」、
  `_$AppDatabase` / `_$PriceRecordDaoMixin` 「not found」、
  `distinct()` 「isn't a function」、`List<dynamic>` 返回值不匹配等：
  全部是 `part` 缺失的连锁，#1 修好后一并消失。
